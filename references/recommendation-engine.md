# 推薦邏輯與決策樹

> 麵魂推薦引擎核心邏輯。根據使用者條件，精準匹配最適合的拉麵店。

## 決策流程圖

```
使用者提問
  │
  ├─ 已提供足夠資訊？ ─── 是 → 直接推薦
  │                         
  └─ 否 → 快問快答（最多5題）
            │
            ├─ Q1: 城市（必問）
            ├─ Q2: 口味偏好（可從對話推斷）
            ├─ Q3: 預算（可選）
            ├─ Q4: 排隊意願（可選）
            └─ Q5: 天氣/情境（可自動判斷）
            │
            └→ 篩選 → 排序 → 推薦 Top 1-3
```

## 篩選規則

### 第一層：城市篩選
- 依使用者所在城市篩選店家
- 若城市無詳細驗證店家，推薦鄰近城市或提供 placeholder 資訊
- 台北市店家最多（280家），推薦時區分中山/大安等商圈

### 第二層：湯頭 × 天氣對照

| 天氣 | 推薦湯頭 | 推薦形式 | 避免 |
|------|----------|----------|------|
| 炎熱（>30°C） | 鹽味、柚香、雞清湯 | 沾麵、冷拉麵 | 濃厚豚骨、味噌 |
| 溫暖（25-30°C） | 雞白湯、醬油、魚介 | 拉麵、沾麵 | — |
| 涼爽（18-25°C） | 全部適合 | 拉麵 | — |
| 寒冷（<18°C） | 豚骨、味噌、擔擔 | 拉麵 | 沾麵、冷麵 |
| 下雨 | 濃厚系、味噌 | 拉麵 | — |

### 第四層：預算篩選

| 預算 | 篩選條件 | 代表 |
|------|----------|------|
| <$200 | price_avg ≤ 200 | 好呷拉麵、平價連鎖 |
| $200-$300 | 200 < price_avg ≤ 300 | 大角、隱家、屯京、RAMEN初 |
| $300+ | price_avg > 300 | 勝王、金龍、麵屋武藏 |
| 沒差 | 不篩選 | — |

### 第五層：排隊篩選

| 排隊意願 | 篩選條件 | 建議 |
|----------|----------|------|
| 不想排 | wait_time = "免排隊" 或 ≤15分 | 大角拉麵、平日午間 |
| 30分鐘內 | wait_time ≤ 30分 | 多數店家平日可達成 |
| 願意等 | 不篩選 | 勝王、鬼金棒等排隊名店 |

## 排序權重

篩選後，依以下權重排序：

| 因素 | 權重 | 說明 |
|------|------|------|
| data_quality | 30% | detailed > placeholder |
| rating | 25% | Google Maps/愛好會評分 |
| 口味匹配度 | 25% | 與使用者偏好的契合度 |
| 排隊友善度 | 10% | 等待時間與使用者意願 |
| 價格匹配度 | 10% | 與預算的契合度 |

## 對話策略

### 使用者只說「推薦拉麵」
→ 回覆：「麵友好！想幫你找到最對味的那碗，先問幾個快問題：你人在哪個城市？偏好濃厚還是清爽？」

### 使用者說「好想吃拉麵」
→ 回覆：「麵魂感應到了！🍜 你在哪邊？我幫你找最近最讚的。」

### 使用者問「XX店好吃嗎？」
→ 查詢店家資料，提供評分/特色/適合族群，給出誠實評價

### 使用者問到資料庫外的店
→ 誠實說明：「這家店不在我目前的資料庫中（目前收錄 1069 家），建議查看 Google Maps 評價或台灣拉麵愛好會的討論。」

### 使用者想比較兩家店
→ 列出對比表格：湯頭/價格/排隊/評分/特色，給出結論

## 進階推薦邏輯

### 「拉麵之旅」規劃
若使用者想一天吃多家，建議：
1. 午餐吃清爽系（鹽味/醬油）→ 不佔胃
2. 下午茶吃沾麵（份量可控）
3. 晚餐吃濃厚系（豚骨/味噌）→ 滿足感

### 季節限定提醒
- 多數創作店家有季節限定品項
- 勝王每週更換限定
- 建議追蹤店家 IG 獲取最新資訊

### 新手→進階→職人 學習路徑

| 階段 | 建議體驗 | 目標 |
|------|----------|------|
| 新手 | 一風堂白丸→隱家雞白湯→屯京豚骨 | 認識四大基礎味型 |
| 進階 | 柑橘清湯→鬼金棒味噌→武藏W湯→元氣屋蝦 | 體驗多元湯頭 |
| 職人 | 勝王限定→金龍淡麗→拉麵公子煮干 | 感受創作與極致 |
